Assignment 11: Northwind Database Activities

Lesson 1 Questions and Answers

  1. **Q1:** What is the primary key of the `Products` table?
    **A1:** The primary key of the `Products` table is `ProductID`.
  2. **Q2:** What is the relationship between the `Orders` and `Customers` tables?
    **A2:** The `Customers` table has a primary key `CustomerID`, which is a foreign key in the `Orders` table.
  3. **Q3:** What is the purpose of the `OrderDetails` table?
    **A3:** The `OrderDetails` table links `Orders` and `Products` with the composite primary key `OrderID` and `ProductID` to store details like quantity and price for each order.
  4. **Q4:** What is the foreign key in the `Orders` table?
    **A4:** The foreign keys in the `Orders` table are `CustomerID` and `EmployeeID`.
  5. **Q5:** How are `Categories` and `Products` related?
    **A5:** The `Categories` table has a primary key `CategoryID`, which is a foreign key in the `Products` table to indicate the category of the product.
  6. **Q6:** Which table contains the supplier information for products?
    **A6:** The `Suppliers` table contains supplier information for products, and the `Products` table references it using the `SupplierID` foreign key.
  7. **Q7:** What does the `Region` table represent?
    **A7:** The `Region` table represents geographic regions and is linked to the `Territories` table through the `RegionID` primary key.
  8. **Q8:** How does the `Employees` table relate to the `Orders` table?
    **A8:** The `Employees` table has a primary key `EmployeeID`, which is a foreign key in the `Orders` table to track which employee handled the order.
  9. **Q9:** What is the purpose of the `Shippers` table?
    **A9:** The `Shippers` table stores information about shipping companies and is linked to the `Orders` table through the `ShipVia` foreign key.
  10. **Q10:** What is the composite primary key of the `OrderDetails` table?
    **A10:** The composite primary key of the `OrderDetails` table is `OrderID` and `ProductID`.
  11. **Q11:** Which fields in the `Products` table refer to other tables?
    **A11:** The fields `SupplierID` (references `Suppliers`) and `CategoryID` (references `Categories`) in the `Products` table refer to other tables.
  12. **Q12:** What is the purpose of the `Territories` table?
    **A12:** The `Territories` table stores information about sales territories and links to the `Region` table through the `RegionID` foreign key.
  13. **Q13:** How are `Employees` and `Territories` related?
    **A13:** The `EmployeeTerritories` table is a junction table that connects `Employees` and `Territories` using the `EmployeeID` and `TerritoryID` fields.
  14. **Q14:** What is the primary key of the `Customers` table?
    **A14:** The primary key of the `Customers` table is `CustomerID`.
  15. **Q15:** How does the `Categories` table contribute to the database structure?
    **A15:** The `Categories` table categorizes products and is referenced in the `Products` table using the `CategoryID` foreign key.

Summary

Completing the Northwind database activities helped me understand the relationships between tables in a database, such as primary and foreign key relationships. I learned how junction tables like `OrderDetails` and `EmployeeTerritories` connect multiple tables and how composite keys work. This exercise emphasized the importance of designing normalized databases for efficient data retrieval and management. These concepts can be applied to real-world database design tasks, ensuring data integrity and improving query performance. Understanding these relationships will help me organize and structure databases effectively in current or future projects.
